cannon — how it works

An AI security harness: fire many Claude agents at a target from different angles, then accumulate → adversarially verify → manage in a ledger → attack-chain → visualize. The local claude CLI does the reasoning; cannon orchestrates, de-dupes, and tracks the lifecycle.

You the operator (CLI / cockpit) cannon the Rust orchestrator claude headless agents (find / verify / threat-model / chain) store results/ + the per-target ledger

The one load-bearing idea

Any single AI scan is flaky and noisy. cannon's design is two countermeasures: union across a permuted salvo (coverage from variance) and a separate adversarial verifier whose only job is to disprove findings (precision). Everything else — the ledger, the cockpit, seeding — is lifecycle management around those two.
