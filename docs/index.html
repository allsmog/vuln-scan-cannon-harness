🜂 cannon

The AI security harness that reasons — not just pattern-matches.

View on GitHub How it works

A single self-contained Rust binary that orchestrates your local, already-authenticated Claude CLI — so your code never leaves your machine. It fires permuted defending-code salvos at a target, then accumulates → adversarially verifies → manages a persistent findings ledger → attack-chains → visualizes, all in your terminal.

# incomplete-fix hunting: re-examine a past security fix for the variant it missed $ cannon permute permdemo --sources commits,threat-intel --budget 0.10 --yes commits +3 proposal(s) ← found the "Fix SQL injection in /search" commit threat-intel +3 proposal(s) ✓ --yes: auto-approved 1 under the cap ▶ P-001 · Variant-hunt around the SQL injection fix (7088da15) ✓ P-001: 3 finding(s) · cost self-calibrated → $0.89/round F-001 CRITICAL SQL injection in /profile endpoint (variant the fix missed) app.py:20

What makes it different

Reasoning over context

Agentic interprocedural taint resolution, a trust-graph reachability oracle, and metamorphic verification — it survives the false-positive traps that kill pattern scanners.

Benchmarked vs Semgrep

An honest head-to-head on OWASP BenchmarkJava with the same scorer — including where reasoning beats dataflow, and where it doesn't.

Signal-driven permutation

Commit archaeology (incl. incomplete-fix variant hunting), threat-model-as-plan, dependency footguns, and bred prompt variants — proposed into a human-gated, cost-capped queue.

Local-first, single binary

No broker, no database, no cloud. Your code stays on your infra; state is plain files on disk.
